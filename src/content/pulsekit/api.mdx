# Pulsekit API
*pulsekit/api*

> API reference for pulsekit — createMonitor, beat, status, services, and isUp.

---

## `createMonitor(options?)`

Create a new monitor.

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `intervalMs` | `number` | `30000` | Max milliseconds between heartbeats before a service is marked down. |

Returns a monitor object with the methods below.

## `monitor.beat(name)`

Record a heartbeat for a service. Registers the service on the first call.

- `name` (`string`) — unique service name.

## `monitor.status(name)`

Get the status of a single service.

Returns `"up"`, `"down"`, or `"unknown"` for services that have never beaten.

## `monitor.isUp(name)`

Convenience boolean that is `true` only when the service status is `"up"`.

- `name` (`string`) — unique service name.

## `monitor.services()`

List all known services with their current status.

```js
monitor.services();
// [{ name: "payments-api", status: "up" }, { name: "worker", status: "down" }]
```
