# Pulsekit Overview

> Tiny heartbeat and uptime monitoring toolkit for Node.js services. Zero dependencies, lazy evaluation, and an opt-in watcher for push-style alerts.

---

Pulsekit is a tiny heartbeat and uptime monitoring toolkit for Node.js services. Each service reports heartbeats to a shared monitor; a service that misses its interval is considered down.

## Install

```bash
npm install pulsekit
```

## Quickstart

```js
import { createMonitor } from "pulsekit";

const monitor = createMonitor({ intervalMs: 30_000, watchIntervalMs: 5_000 });

monitor.watch(({ name, from, to, at }) => {
  if (to === "down") page(`${name} stopped beating`);
});

// Somewhere in each service's loop:
monitor.beat("payments-api");
monitor.beat("nightly-batch", { intervalMs: 90 * 60_000 }); // its own cadence

// Later:
monitor.status("payments-api"); // "up" | "down" | "unknown"
monitor.summary(); // { up: 2, down: 0, total: 2 }
```

## How it works

- A **monitor** tracks named services in memory.
- Calling `beat(name)` registers the service on first call and refreshes its
  last-seen timestamp on every call.
- A service whose last heartbeat is older than `intervalMs` reports as
  `"down"`; unregistered services report `"unknown"`.
- Status is evaluated **lazily** on read — an idle monitor costs nothing.
- Set `watchIntervalMs` to opt into automatic transition detection: the monitor
  sweeps periodically and notifies `watch()` listeners when services go down.
  The sweep timer is unref'd and never keeps the process alive.
- Each service can have its own `intervalMs`, so a slow nightly batch can
  coexist with fast API heartbeats.

See the [API reference](/pulsekit/api) for the full surface.
