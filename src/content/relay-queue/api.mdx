Create and control in-memory job queues.

## Creating a queue

### `createQueue(options?)`

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `concurrency` | `number` | `1` | Max jobs running at once. |
| `retries` | `number` | `0` | Retry attempts per job after the first failure. |
| `timeoutMs` | `number` | — | Per-attempt time limit; a timed-out attempt counts as a failure and consumes a retry. |
| `retryDelayMs` | `number` | `0` | Delay before the first retry. |
| `backoffFactor` | `number` | `2` | Multiplier applied to the delay on each subsequent retry. |
| `onError` | `(err, jobId) => void` | no-op | Called when a job exhausts its retries. Timeouts surface as `TimeoutError`. |

## Adding work

### `queue.add(fn, options?)`

Add a job to the queue.

- `fn` (`() => Promise<any>`) — async job function.
- `options.priority` (`number`, default `0`) — higher runs first; FIFO
  within a priority level.
- `options.timeoutMs` (`number`) — per-attempt limit for this job,
  overriding the queue default.
- Returns a numeric job id, usable for correlating `onError` reports.

### `queue.addAll(fns)`

Enqueue several jobs at once; returns their ids in input order.

## Observing

### `queue.size()`

Number of jobs waiting to run (running jobs are not counted).

### `queue.running()`

Number of jobs currently in flight. A job waiting on a retry delay still
counts as running.

### `queue.stats()`

Lifetime counters plus current state:

```js
queue.stats();
// { added: 12, completed: 9, failed: 1, retried: 4, timedOut: 2,
//   running: 1, pending: 1 }
```

### `queue.onIdle()`

Returns a promise that resolves when the queue is fully drained — no running
jobs, no waiting jobs, and no pending retry delays.

## Controlling

### `queue.pause()` / `queue.resume()`

Stop and restart job starts; running jobs always finish.

### `queue.isPaused()`

Whether the queue is currently paused.

### `queue.clear()`

Drop all waiting jobs without running them; returns the number dropped.
Running jobs finish normally.
