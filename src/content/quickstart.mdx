In five minutes you'll have a small worker process that queues jobs with
retries, logs everything it does without leaking secrets, and reports a
heartbeat you can alert on. That's the whole toolkit in one file.

## Prerequisites

- Node.js 18 or newer
- A project with `"type": "module"` in its `package.json` (all three
  libraries are ES modules)

<Steps>
  <Step title="Install the libraries">
    Install whichever you need — they have no dependencies and no opinions
    about each other:

    ```bash
    npm install pulsekit quill-log relay-queue
    ```
  </Step>
  <Step title="Create a logger">
    Structured JSON lines, with anything sensitive masked before it can reach
    a sink:

    ```js
    import { createLogger } from "quill-log";

    const log = createLogger({
      base: { service: "email-worker" },
      redact: ["password", "token", "apiKey"],
    });
    ```
  </Step>
  <Step title="Create a queue">
    Two jobs at a time, three retries with exponential backoff, ten-second
    attempts:

    ```js
    import { createQueue } from "relay-queue";

    const queue = createQueue({
      concurrency: 2,
      retries: 3,
      retryDelayMs: 250,
      timeoutMs: 10_000,
      onError: (err, jobId) => log.error("job failed for good", { err, jobId }),
    });
    ```
  </Step>
  <Step title="Add a heartbeat">
    Beat every time the worker completes a loop; alert when it stops:

    ```js
    import { createMonitor } from "pulsekit";

    const monitor = createMonitor({ intervalMs: 30_000, watchIntervalMs: 5_000 });
    monitor.watch(({ name, to }) => {
      if (to === "down") log.warn("service stopped beating", { name });
    });
    ```
  </Step>
  <Step title="Put it together">
    ```js
    async function processEmails() {
      const end = log.time("email batch");
      const batch = await fetchPendingEmails();

      queue.addAll(batch.map((email) => () => sendEmail(email)));
      await queue.onIdle();

      end({ sent: queue.stats().completed });
      monitor.beat("email-worker");
    }

    setInterval(processEmails, 15_000);
    ```

    Run it and watch the records flow:

    ```json
    {"level":"info","msg":"email batch","time":"2026-07-23T09:14:02.113Z","service":"email-worker","durationMs":1873,"sent":42}
    ```
  </Step>
</Steps>
