## Why these three

Every long-running Node.js service ends up rebuilding the same plumbing: a
health signal for the load balancer, logs that don't leak credentials, and a
queue that retries flaky work without hammering a struggling dependency. Each
library here does one of those jobs with a deliberately small surface:

- **No dependencies.** Each package is a single ES module you can read in one
  sitting. Nothing to audit but the code itself.
- **Lazy by default.** Idle monitors, filtered log levels, and paused queues
  cost nothing. You opt into timers; none of them will hold your process open.
- **Test-friendly.** Injectable clocks and formatter sinks mean the test
  suites never sleep and never spy on stdout.

They compose naturally — [Using the libraries together](/together) walks
through an instrumented worker that uses all three — but each stands alone.
